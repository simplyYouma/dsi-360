# Alignement ITIL — SI-12.02 (Service Desk), SI-12.03 (Problèmes), SI-12.05 (Demandes)

Synthèse : ces trois procédures sont **en grande partie couvertes** par les mécanismes déjà en
place. Points de couverture et écarts résiduels.

## SI-12.02 — Service Desk
- **Point d'entrée unique** : la plateforme centralise incidents et demandes → ✅.
- **Support N1 → N2 → N3** : le niveau est **déduit du gestionnaire** que porte l'import quotidien,
  N3 = DBS ; l'escalade elle-même se fait hors du système → ✅ (cf. SI-12.01, ADR-0005).
- **Catégorisation** : catégories paramétrables par module (admin) → ✅.
- Résiduel : indicateurs Service Desk dédiés (volumétrie par niveau) — via Analyses, à enrichir.

## SI-12.03 — Problèmes
- Le module `probleme` a un **cycle de vie** défini (Nouveau → En cours d'analyse → Résolu →
  Clôturé, + Réouvert) dans `domain/etats.py`, mais **n'est pas exposé** en tant que module
  utilisateur (absent des 9 modules du cahier des charges).
- Décision : laissé **non exposé** pour l'instant (concept ITIL interne). Activable ultérieurement
  (routeur + page) sans refonte, la machine à états existant déjà.

## SI-12.05 — Demandes de service
- **Workflow de validation** : le cycle Nouvelle → Qualifiée → En cours → **En validation** →
  Résolue → Clôturée porte l'étape d'approbation → ✅.
- **Demandes nécessitant validation/approbation** (accès sensibles, achats, prestations à coût) :
  s'appuie sur les **Valideurs** de l'activité (désormais câblés) + l'état « En validation » → ✅
  (mécanisme en place ; le caractère bloquant de l'approbation reste à durcir si souhaité).
- **Types de demande** (création de compte, habilitation, VPN, matériel, assistance…) : portés par
  les catégories du module `demande` → ✅.
- **SLA** : paramétrable par (module, priorité) ; le SLA spécifique d'un service du catalogue
  primerait — le **Catalogue de services** (référentiel de services + SLA dédiés) n'est pas
  implémenté → 🟡 (Phase ultérieure).
- **Escalade vers Changement / Projet** pour les demandes hors-catalogue impliquant un
  développement : lien manuel possible ; automatisation → 🟡 (Phase ultérieure).

## Reste à décider (transverse)
- **Approbation bloquante** : conditionner le passage « Validé / En validation → suite » à
  l'approbation explicite des valideurs (aujourd'hui les valideurs sont désignés mais
  n'« approuvent » pas formellement).
- **Catalogue de services** (SI-12.05) et **catalogue des changements standards** (SI-12.04).
